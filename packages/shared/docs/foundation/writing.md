# UI 문구 작성

버튼·에러·빈 상태 등 구체 문구 규칙. 말투 원칙은 [voice-and-tone.md](./voice-and-tone.md), 숫자·단위 표기는 [international-design.md](./international-design.md).

## 버튼 = 동사구

- 그 버튼이 하는 **행동**을 쓴다: "담기", "주문하기", "저장", "쿠폰 적용".
- 모호한 "확인"·"예"·"아니오" 금지 — 무엇을 확인하는지 알 수 없다.
- 파괴적 액션 다이얼로그: 구체 동사 버튼("삭제") + "취소". "예/아니오"로 묻지 않는다.
  - 예: "이 상품을 삭제할까요?" → [삭제] [취소]

## 에러 메시지 = 원인 + 해결 행동

- 무엇이 왜 안 됐고, 다음에 뭘 하면 되는지.
- 예: "재고가 없어요. 다른 옵션을 선택해 주세요." / "결제가 취소됐어요. 카드 정보를 확인하고 다시 시도해 주세요."
- 기술 용어·에러 코드 노출 금지(로그로). 사용자 탓 금지. → [voice-and-tone.md](./voice-and-tone.md)
- 동적 에러는 `aria-live`로 알린다. → [inclusive-design.md](./inclusive-design.md)

## 빈 상태 = 다음 행동 제안

- "아직 없어요"에서 멈추지 말고 다음 행동을 준다.
- 예: "찜한 상품이 없어요. 마음에 드는 상품을 찜해 보세요." + [상품 둘러보기].

## 플레이스홀더

- **예시로만** 쓴다: "예: 홍길동", "예: 010-1234-5678".
- 레이블을 대체하지 않는다 — 입력에는 항상 보이는 `<label>`이 있어야 한다. → [inclusive-design.md](./inclusive-design.md)

## 숫자·단위

- 통화 `₩` + 천단위 콤마, 날짜 `YYYY.MM.DD`, 상대시간 `n분 전`. 규칙은 [international-design.md](./international-design.md).
