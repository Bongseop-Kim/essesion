# 보이스 & 톤

store·admin의 말투 원칙. 구체적 UI 문구 규칙은 [writing.md](./writing.md).

## store (고객 대상)

- **해요체.** 친근하고 정중하게("배송이 시작됐어요").
- 짧고 명확하게. 한 문장 한 목적.
- 결제·배송·환불 등 민감한 순간은 **안심시키는 톤**("결제 정보는 안전하게 보관돼요").
- 과장·낚시 금지("역대급", "단 하나 남은!" 지양). 사실만.

## admin (운영자 대상)

- **명사형 종결 허용**("주문 3건 처리 완료", "재고 부족"). 정보 밀도 우선.
- 기능·객체 명칭을 **일관되게**(같은 대상을 "주문"/"오더"로 섞지 않음).
- 간결하게 — 운영자는 반복 작업자다. 설명보다 라벨.

## 공통

- **사용자 탓 금지.** "잘못 입력했습니다" ✗ → "이메일 형식을 확인해 주세요" ✓.
- **능동태.** "저장되었습니다" 보다 "저장했어요"(store) / "저장 완료"(admin).
- 한 문장 한 목적. 부정문보다 긍정문.
- 숫자·통화·날짜 표기는 [international-design.md](./international-design.md).
